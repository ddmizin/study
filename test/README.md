# Лабораторная работа 7

Работа с внешними библиотеками

## Задача

Спроектировать и реализовать консольное приложение для нахождения маршрутов из Санкт-Петербурга до вашего родного города, используя Yandex API, внешние библиотеки и кэширование.

## Требования

Консольное приложение должно искать список возможных маршрутов из Санкт-Петербурга до вашего родного города (в случае, если вы из Санкт-Петербурга — до Пскова) и обратно, с не более чем одной пересадкой любыми видами транспорта (например, самолёт + поезд) на определённую дату (передаётся аргументом командной строки).  
Необходимо кэшировать найденные промежуточные результаты для уменьшения количества запросов к внешнему сервису.

Программа должна иметь интерактивный режим, уметь обрабатывать более 1 запроса пользователя за 1 запуск. 

## Источник данных

В качестве источника информации должно быть использовано [API Яндекс Расписаний](https://yandex.ru/dev/rasp/doc/ru/).  
Для разработчиков оно ограничено 500 запросами в день. Стоит внимательно изучить API, продумать, какая информация является неизменной, что можно сохранить локально и переиспользовать.

## Реализация

Работа с API предполагает выполнение [HTTP-запросов](https://en.wikipedia.org/wiki/HTTP).  
Для этого необходимо воспользоваться библиотекой [C++ Requests](https://github.com/libcpr/cpr). Используйте версию `>=1.11.0`.
В качестве библиотеки для работы с JSON-ответом используйте [nlohmann/json](https://github.com/nlohmann/json).

При взаимодействии с внешними сервисами может возникать достаточно большое количество коллизий и краевых случаев.  
Внимательно продумайте это! Ваша программа должна корректно работать и не "падать".

## Кэширование

В данной работе необходимо реализовать [кэширование](https://en.wikipedia.org/wiki/Cache_replacement_policies) ответов Яндекс Расписаний для уменьшения нагрузки и выполнения ограничений со стороны сервиса.  
Алгоритм кэширования должен ограничивать размер кэша и время жизни данных в нём (в противном случае вы будете показывать неактуальную информацию).  
Конкретный алгоритм остаётся на ваше усмотрение.

## Тестирование

В данной работе тесты являются частью задания.
Сетевые походы при этом можно [замокать](https://en.wikipedia.org/wiki/Mock_object) с помощь [gmock](https://google.github.io/googletest/gmock_cook_book.html)
